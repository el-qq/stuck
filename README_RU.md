STUCK — read-only веб-инструмент, который объясняет, как запрос к домену,
IP-адресу или `host:port` проходит через Ideco NGFW Novum. Приложение загружает
текущую конфигурацию NGFW, локально проверяет упорядоченный конвейер обработки и
показывает первое подходящее правило и результат каждой стадии.

Проверку можно выполнить без конкретного пользователя или от лица пользователя
NGFW. Во втором случае учитываются активные сессии и назначенные IP-привязки.
Если IP неизвестен, правила пользователя и групп всё равно проверяются, а
зависящие от IP решения честно остаются неопределёнными.

## Ключевые возможности ✨

- **🔎 Трассировка трафика** — проверяйте домен, IP-адрес или `host:port`;
  пресеты сервисов и портов ускоряют типовые проверки. Выбор пользователя NGFW
  и, когда он известен, IP-адреса источника добавляет контекст пользователя.
- **🧭 Просмотр пути обработки** — следите за решениями по порядку: аппаратная
  фильтрация, DNS/NAT, межсетевой экран и модули инспекции. Недостающий
  контекст пакета или политики отображается как `unknown`, а не считается
  разрешённым.
- **🧹 Гигиена правил** — находите потенциально перекрытые, избыточные,
  недостижимые и слишком широкие правила firewall. Это рекомендательные
  выводы: STUCK не меняет конфигурацию NGFW.
- **🔒 Безопасные экспорты** — скачивайте JSON одной трассировки или
  отформатированный снимок правил. Экспорт правил требует подтверждения и
  удаляет отображаемые данные пользователя и комментарии; в экспорте
  трассировки остаётся технический идентификатор пользователя.
- **🛡️ Вход и дальнейшая проверка** — поддерживаются разрешённые роли
  администратора NGFW и ввод второго фактора, а из трассировки можно перейти
  к соответствующему разделу правил в NGFW.

## Основные свойства

- Backend приложения не изменяет конфигурацию NGFW.
- Пароли и cookie NGFW остаются только на backend.
- Снимки правил изолированы по связке `администратор + NGFW` и хранятся в памяти
  до ручного обновления или перезапуска.
- Frontend написан на React, Vite и TypeScript; backend — на FastAPI.
- Есть адаптивный UI, локализация, офлайн-демо и диагностический экспорт правил.
- Отдельная административная утилита может создать префиксные тестовые данные;
  она не используется read-only приложением.

## Быстрый запуск

Нужны Python, Node.js, npm и POSIX shell. Docker опционален.

```bash
npm run setup
npm start
```

Откройте <http://localhost:3000>. `Ctrl+C` останавливает оба dev-сервиса. Эта
команда явно включает неограниченный лабораторный режим NGFW и привязывает
backend к localhost; на форме входа показывается предупреждение.

Для обычного запуска через Docker владелец установки задаёт собственный
allowlist NGFW:

```bash
STUCK_ALLOWED_NGFW_HOSTS=ngfw.company.local,192.168.10.1 \
  docker compose up --build
```

После запуска откройте <https://localhost>.
Для постоянной установки скопируйте `.env.example` в `.env`, замените пример
NGFW своими адресами и запускайте Compose без префикса переменных.

Только для изолированной лаборатории ограничение можно отключить явно:

```bash
STUCK_ALLOW_ANY_NGFW=true docker compose up --build
```

Конфигурация хранится в [backend/conf/stuck.conf](backend/conf/stuck.conf), а
одноимённые переменные окружения имеют приоритет. Полные разрешённые графы
зависимостей хранятся в lock-файлах; прямые frontend-версии также точно указаны
в `package.json`, а backend-файлы `.in` остаются входом для обновления.

Вне `npm start` доступ к NGFW работает по принципу fail-closed. Настройте одно
или оба значения:

```env
STUCK_ALLOWED_NGFW_HOSTS=ngfw.company.local,192.168.10.1
STUCK_ALLOWED_NGFW_CIDRS=10.20.0.0/16
```

`STUCK_ALLOW_ANY_NGFW=true` — только явный режим лаборатории. Loopback,
link-local, multicast и unspecified адреса запрещены во всех режимах.

## Конфигурация

Задайте значения в `backend/conf/stuck.conf` или одноимёнными переменными
окружения: переменные окружения имеют приоритет.

| Параметр                       | Значение по умолчанию | Назначение                                                                                                                                                 |
| ------------------------------ | --------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `STUCK_DEFAULT_SERVER`         | пусто                 | Необязательный NGFW-хост, фиксируемый в форме входа и API; переменная окружения имеет приоритет над конфигом.                                              |
| `STUCK_NGFW_PORT`              | `8443`                | HTTPS-порт подключения к NGFW.                                                                                                                             |
| `STUCK_ALLOWED_NGFW_HOSTS`     | пусто                 | Точный список разрешённых NGFW-хостов или IPv4-адресов через запятую.                                                                                      |
| `STUCK_ALLOWED_NGFW_CIDRS`     | пусто                 | Список разрешённых IPv4- или IPv6-сетей через запятую.                                                                                                     |
| `STUCK_ALLOW_ANY_NGFW`         | `false`               | Явно разрешает любой безопасный NGFW-хост; только для лаборатории.                                                                                         |
| `STUCK_SESSION_TTL_HOURS`      | `10`                  | Время жизни browser-сессии STUCK в часах.                                                                                                                  |
| `STUCK_2FA_TTL_SECONDS`        | `180`                 | Время жизни 2FA-челленджа — сколько форма ввода кода действительна после ввода пароля. Диапазон 30–600.                                                    |
| `STUCK_COOKIE_SECURE`          | `true`                | Помечает cookie сессии как доступную только по HTTPS.                                                                                                      |
| `STUCK_COOKIE_SAMESITE`        | `lax`                 | Политика SameSite для cookie сессии.                                                                                                                       |
| `STUCK_NGFW_VERIFY_TLS`        | `false`               | Проверять TLS-сертификат NGFW.                                                                                                                             |
| `STUCK_NGFW_CA_BUNDLE`         | пусто                 | Путь к набору CA при включённой проверке TLS.                                                                                                              |
| `STUCK_ALLOWED_ORIGINS`        | `https://localhost`   | Разрешённые CORS-origin через запятую для разработки или раздельного развёртывания.                                                                        |
| `STUCK_BACKEND_PORT`           | `8000`                | Порт, на котором слушает backend.                                                                                                                          |
| `STUCK_TRACE_DEFAULT_PORT`     | `443`                 | Порт назначения, если проверяемый адрес не содержит порт.                                                                                                  |
| `STUCK_NGFW_TIMEOUT_SECONDS`   | `15`                  | Таймаут одного запроса к NGFW в секундах.                                                                                                                  |
| `STUCK_LOG_LEVEL`              | `INFO`                | Минимальный уровень лога: `DEBUG`, `INFO`, `WARNING` или `ERROR`.                                                                                          |
| `STUCK_LOG_FORMAT`             | `text`                | Формат логов: `text` или `json`.                                                                                                                           |
| `STUCK_LOG_FILE`               | пусто                 | Путь к файлу лога; пустое значение выводит логи в stdout.                                                                                                  |
| `STUCK_ENABLE_RULES_EXPORT`    | `true`                | Разрешает экспорт снимка правил для авторизованного пользователя.                                                                                          |
| `STUCK_ENABLE_TRACE_ANIMATION` | `true`                | Поочерёдно показывает этапы трассировки на desktop и кнопку **«Пропустить анимацию»**; `false` сразу показывает весь результат. Действует и в офлайн-демо. |
| `STUCK_ENABLE_RULE_HYGIENE`    | `true`                | Включает read-only панель «Гигиена правил» (затенённые / избыточные / недостижимые / слишком широкие правила МЭ); `false` — ответ 404 и панель скрыта.     |
| `STUCK_REQUIRE_READONLY_ADMIN` | `false`               | Разрешает вход только администраторам NGFW с ролью «только чтение»; любая другая роль отклоняется после аутентификации (включая 2FA) с `readonly_admin_required`. |

Чтобы зафиксировать один NGFW-хост при локальном запуске, используйте:

```bash
STUCK_DEFAULT_SERVER=ngfw.example.local npm start
```

Для Docker Compose укажите значение в `.env` или передайте его командой:

```bash
STUCK_DEFAULT_SERVER=ngfw.example.local \
  STUCK_ALLOWED_NGFW_HOSTS=ngfw.example.local \
  docker compose up --build
```

Значение должно быть именем хоста или IPv4-адресом без схемы, порта и пути.
Непустое значение фиксирует сервер и в форме входа, и в API.

IBM Plex поставляется локально из точно зафиксированных npm-пакетов: готовый UI
не обращается к Google Fonts или другому CDN. Лицензия шрифта входит в сборку.

Базовые контейнеры зафиксированы одновременно читаемым patch-тегом и
неизменяемым multi-platform digest. Это выбранный режим для релизов: одинаковая
ревизия собирается на поддерживаемых архитектурах, а обновление образа является
явным изменением в git. Варианты с `major/minor`-тегом автоматически получают
патчи, но повторная сборка может незаметно измениться; один patch-тег понятнее,
но всё ещё изменяем владельцем registry. При обновлении образа проверьте новый
digest через `docker buildx imagetools inspect <image>:<tag>`, замените tag и
digest вместе, затем пересоберите с `docker compose build --pull`.

## CI

Каждый pull request (и каждый push в `main`) запускает пайплайн GitHub Actions
[.github/workflows/pr.yml](.github/workflows/pr.yml):
`lint` → `backend` + `frontend` (параллельно) → `e2e` + `docker` + `audit`
(параллельно).

- `lint` — ruff (линт и проверка формата), Prettier, EditorConfig и
  `version:check`.
- `backend` — pytest: тесты приложения плюс набор `tools/ngfw_testdata`.
- `frontend` — проверка типов TypeScript и vitest.
- `e2e` — браузерные тесты Playwright (Chromium).
- `docker` — сборка обоих образов контейнеров.
- `audit` — pip-audit и npm audit; только информирующий, PR не блокирует.

## Ограничения

- Сессии и снимки правил хранятся в памяти процесса и очищаются при рестарте.
- Антивирус, IPS и условия, недоступные через read-only API, нельзя полностью
  смоделировать без реального трафика; UI показывает условный или неизвестный
  результат.
- Двухфакторный вход администратора завершается через WebSocket-челлендж
  NGFW: после ввода пароля STUCK показывает форму для кода подтверждения.

## Отказ от ответственности

STUCK — независимый диагностический инструмент, а не официальный продукт Ideco.
Его результат основан на доступных read-only данных и не гарантирует фактическое
поведение трафика. Важные решения проверяйте на целевом NGFW, а административную
утилиту используйте только на системах, где вам разрешено менять конфигурацию.
